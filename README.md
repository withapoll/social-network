# Заготовка для социальной сети

Базовый скелет социальной сети для ДЗ курса «Разработка и проектирование
высоконагруженных систем». Реализованы регистрация пользователя, простейшая
авторизация и просмотр анкеты по ID.

## Что внутри

- **Язык/фреймворк:** Python 3.11+ / FastAPI
- **БД:** PostgreSQL, доступ через `psycopg` 3 — **чистый параметризованный SQL, без ORM**
- **Архитектура:** монолитное приложение
- **Пароли:** хранятся в виде bcrypt-хэша
- **Авторизация:** токен выдаётся при `/login`, передаётся в заголовке `Authorization: Bearer <token>`
- **Фронтенд:** минимальные статические страницы (HTML/CSS/JS) в каталоге `static/`

Сознательно **не используются** (по требованию ДЗ): репликация, шардирование,
дополнительные индексы, кэширование.

## Структура проекта

```
social-network/
├── app/
│   ├── config.py        — загрузка настроек из .env
│   ├── db.py            — пул соединений PostgreSQL
│   ├── security.py      — bcrypt-хэширование паролей, генерация токенов
│   ├── models.py        — Pydantic-схемы запросов/ответов
│   ├── repository.py    — слой доступа к данным (чистый SQL)
│   └── main.py          — FastAPI-приложение и эндпоинты
├── static/              — минимальный фронтенд
├── sql/schema.sql       — схема БД
├── requirements.txt
├── .env.example
└── postman_collection.json
```

## Эндпоинты

| Метод | Путь                | Описание                              | Авторизация |
|-------|---------------------|---------------------------------------|-------------|
| POST  | `/user/register`    | Регистрация пользователя              | нет         |
| POST  | `/login`            | Авторизация, выдаёт токен             | нет         |
| GET   | `/user/get/{id}`    | Анкета пользователя по ID             | да          |
| GET   | `/health`           | Проверка живости сервиса              | нет         |
| GET   | `/`                 | Статический фронтенд                  | нет         |

Интерактивная документация (Swagger UI) доступна на `http://localhost:8000/docs`.

## Локальный запуск (без Docker)

### 1. Установить PostgreSQL

Linux (Debian/Ubuntu):
```bash
sudo apt update && sudo apt install -y postgresql
sudo service postgresql start
```
Windows / macOS — установить с https://www.postgresql.org/download/.

### 2. Создать базу данных и пользователя

```bash
sudo -u postgres psql <<'SQL'
CREATE USER social WITH PASSWORD 'social';
CREATE DATABASE social_network OWNER social;
SQL
```

### 3. Применить схему

```bash
psql -h localhost -U social -d social_network -f sql/schema.sql
```

### 4. Настроить окружение

```bash
cp .env.example .env
# при необходимости отредактируйте параметры подключения в .env
```

### 5. Установить зависимости и запустить

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python -m app.main
```

Приложение поднимется на `http://localhost:8000`.
Фронтенд — на корневом URL, Swagger UI — на `/docs`.

## Проверка через Postman

Импортируйте `postman_collection.json`. Запросы выполняются по порядку:

1. **Register** — создаёт пользователя, его `user_id` автоматически
   сохраняется в переменную коллекции.
2. **Login** — авторизуется этим `user_id`, полученный `token` автоматически
   сохраняется в переменную коллекции.
3. **Get user by id** — получает анкету (использует сохранённый токен).
4. **Get user by id — без токена** — проверяет, что без токена возвращается 401.
5. **Health** — проверка живости.

Базовый URL задаётся переменной коллекции `base_url` (по умолчанию
`http://localhost:8000`).

## Безопасность

- **SQL-инъекции:** все запросы в `app/repository.py` параметризованы — значения
  передаются отдельным аргументом в `cur.execute(...)`, строки запросов
  никогда не собираются конкатенацией с пользовательскими данными.
- **Хранение паролей:** пароль никогда не сохраняется в открытом виде — в БД
  лежит только bcrypt-хэш со случайной солью (`app/security.py`).
- **Токены:** генерируются криптостойким `secrets.token_hex`.
